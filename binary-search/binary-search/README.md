# Binary Search

## Условие

Дан отсортированный массив `nums` и число `target`. Найди индекс `target` в массиве. Если не найден — верни `-1`. Сложность должна быть O(log n).

## Ключевая идея

Каждый раз делим массив пополам. Смотрим на средний элемент — если он больше `target`, ищем в левой половине, если меньше — в правой. За каждый шаг отбрасываем половину оставшихся элементов.


## Подводный камень с mid

```go
mid := (lo + hi) / 2      // опасно: lo+hi может переполнить int32
mid := lo + (hi-lo) / 2   // правильно: разность всегда в пределах
```

В Go `int` на 64-битных системах 64-битный, поэтому переполнение маловероятно — но формулу принято писать безопасную.

## Сложность

| | Время | Память |
|---|---|---|
| Двоичный поиск | O(log n) | O(1) |

На массиве из миллиона элементов — максимум 20 итераций.

## Граничные случаи

- Пустой массив → `lo=0, hi=-1`, цикл не входит, возвращаем `-1`
- Один элемент — работает
- `target` меньше минимума или больше максимума — диапазон схлопнется и вернём `-1`
