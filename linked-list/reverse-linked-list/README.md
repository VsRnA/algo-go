# Reverse Linked List

## Условие

Дан односвязный список. Разверни его и верни новую голову.

```
вход:  1 → 2 → 3 → 4 → 5 → nil
выход: 5 → 4 → 3 → 2 → 1 → nil
```

## Ключевая идея

Идём по списку и на каждом шаге перенаправляем указатель `Next` текущего узла на предыдущий. Главное — сохранить следующий узел до того, как перезапишем стрелку, иначе потеряем остаток списка.

## Сложность

|                | Время | Память              |
|----------------|-------|---------------------|
| Итеративное    | O(n)  | O(1)                |
| Рекурсивное    | O(n)  | O(n) — стек вызовов |

На интервью предпочтительнее итеративное — O(1) по памяти и нет риска stackoverflow на длинном списке.

## Граничные случаи

- Пустой список `nil` → вернуть `nil`
- Один элемент → вернуть его же
- Два элемента → проверь что обе стрелки корректно переставлены
